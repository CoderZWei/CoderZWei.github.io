Android开发调试相关

Posted on 2019-05-25 | In 技术

快捷键：

1）Ctrl+F12：查看类中的所有变量、方法、内部类、内部接口。

2）Ctrl+Tab：切换面板或文件，类似于Windows下的Alt+Tab

3）Ctrl + N：快速查找类

4）Ctrl+R：在当前文件中替换输入的内容

5）Ctrl+Shift+R：全局替换

6）Ctrl+Shift+F 或 Double Shift：全局搜索。Double Shift 搜索的范围更广，支持符号检索，除了Ctrl + N、Ctrl + Shift + N的检索内容外，还可以搜索到变量、资源id等

7）Shift+F6：快速重命名。选中某个类、变量、资源id等之后，可以快速重命名，只要改动一个位置，其它地方也都会自动全部重命名。

8）Alt+F7：快速查找某个类、方法、变量、资源id被调用的地方

9）Ctrl+D：快速复制行

10）Alt+Insert：快速插入代码。可以快速生成构造方法、Getter/Setter方法等

11）Alt+Enter：快速修复错误

12）Ctrl+Alt+Space：类名或接口名提示。输入一个不完整的类名或者接口名，按Ctrl + Alt + Space，会给出完整类名或接口名的提示

13）Ctrl + Shift + I：快速查看某个方法、类、接口的内容。将鼠标光标定位到某个方法、类名、接口名，然后按Ctrl + Shift + I，将会在当前光标位置显示改方法、类、接口的内容

14）Ctrl+P：显示参数信息

15）Ctrl+/：单行注释

16）Ctrl+Shift+/：多行注释

17）Ctrl+Alt+O：优化导入的类和包

18）Ctrl+F12：显示当前文件的结构

19）Ctrl+H：显示类结构图

git相关

Posted on 2019-05-17 | In 技术

创建分支（dev）：

git branch dev

切换分支：

git checkout dev

在本地创建新的分支并切换到此分支：

git checkout -b dev //git branch dev + git checkout dev

将代码从仓库提交到远程库（不需要事先在远程库创建分支）：

git push origin dev

查看本地分支：

git branch

创建远程库的dev分支到本地：

git checkout -b dev origin/dev

将dev分支合并到某分支

切换到目标分支，然后

git merge dev

删除dev分支：

git branch -d dev

在远程库创建仓库时有readme等文件时，本地需要和远程库同步

1）在本地git init

2）git remote add origin git@github.com:XXX/XXX.git 进行连接

3）git pull origin master 将远程库文件拉下来

4）git add

5）git commit

6）git push -u origin master

变基(rebase)操作：

git add XXX

git commit XXX

git rebase 分支名

git push origin XXX

merge和rebase区别：

处理冲突的方式：

  • （一股脑）使用merge命令合并分支，解决完冲突，执行git add .和git commit -m'fix conflict'。这个时候会产生一个commit。

  • （交互式）使用rebase命令合并分支，解决完冲突，执行git add .和git rebase --continue，不会产生额外的commit。这样的好处是，‘干净’，分支上不会有无意义的解决分支的commit；坏处，如果合并的分支中存在多个commit，需要重复处理多次冲突。

    注意：

    1）不要在公共分支使用rebase

    2）本地和远端对应同一条分支,优先使用rebase,而不是merge
