git相关

Posted on 2019-05-17

创建分支（dev）：

git branch dev

切换分支：

git checkout dev

在本地创建新的分支并切换到此分支：

git checkout -b dev //git branch dev + git checkout dev

将代码从仓库提交到远程库（不需要事先在远程库创建分支）：

git push origin dev

查看本地分支：

git branch

创建远程库的dev分支到本地：

git checkout -b dev origin/dev

将dev分支合并到某分支

切换到目标分支，然后

git merge dev

删除dev分支：

git branch -d dev

在远程库创建仓库时有readme等文件时，本地需要和远程库同步

1）在本地git init

2）git remote add origin git@github.com:XXX/XXX.git 进行连接

3）git pull origin master 将远程库文件拉下来

4）git add

5）git commit

6）git push -u origin master

变基(rebase)操作：

git add XXX

git commit XXX

git rebase 分支名

git push origin XXX

merge和rebase区别：

处理冲突的方式：

  • （一股脑）使用merge命令合并分支，解决完冲突，执行git add .和git commit -m'fix conflict'。这个时候会产生一个commit。

  • （交互式）使用rebase命令合并分支，解决完冲突，执行git add .和git rebase --continue，不会产生额外的commit。这样的好处是，‘干净’，分支上不会有无意义的解决分支的commit；坏处，如果合并的分支中存在多个commit，需要重复处理多次冲突。

    注意：

    1）不要在公共分支使用rebase

    2）本地和远端对应同一条分支,优先使用rebase,而不是merge
